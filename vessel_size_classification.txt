vessel_size_classification

Short Summary — Vessel Size Classification Map

This file defines the official VK Charts vessel size categories for all dry-cargo vessels (bulk + general cargo).
It is the only source of truth the AI must use when determining what “size class” a vessel belongs to based on its DWT.

The document contains:

7 main size groups (Small Coasters → Panamax)

Precise DWT ranges, category codes, and examples

Size-specific economics (ballast tolerance, operating cost)

Typical trade patterns for each size

Penalties and bonuses for proximity scoring based on size

Recommended vessel sizes for different cargo types

Rules for how size affects scoring inside the AI logic

This file is directly linked to the Proximity Scoring Matrix, because proximity penalties/bonuses depend heavily on vessel size.

▶️ How to Use This File — Short Instructions
1. Determine vessel size category by DWT

AI must always use the DWT ranges defined here:

0–4000      → small_coasters  
4001–17000  → coasters  
17001–24000 → small_handy  
24001–40000 → handy  
40001–50000 → small_supramax  
50001–65000 → supramax  
65000–80000 → panamax


This classification must never be guessed — only taken from this file.

2. Apply size-specific economic behavior

Each size class defines:

ballast distance tolerance

how aggressively a vessel seeks backhaul

typical operational cost

realistic trade area

This affects:

proximity penalties

regional pattern scoring

cargo–vessel compatibility

3. Use the size category inside proximity scoring

The Proximity Scoring Matrix uses distance penalties that scale with vessel size.

Examples:

small coasters → very high penalty per 100 nm

coasters → high

small handy → medium

handy → medium/low

supramax/panamax → low

AI must always cross-reference this file when calculating proximity.

4. Use size match rules in cargo compatibility

The file defines the correct vessel sizes for typical cargo flows:

Black Sea → Med grain → coasters / small handy

Black Sea → Far East grain → supramax / panamax

Steel products → coasters / small handy

Fertilizers → coasters / small handy

AI should apply these as scoring modifiers when evaluating vessel–cargo match.

5. Reject incorrect vessel types

The file clearly states the scope:

✔ dry cargo bulk
✔ general cargo (bulk ops)
✘ tankers
✘ containers
✘ RoRo
✘ LPG/LNG
✘ non-bulk breakbulk

If the vessel doesn't fall into the scope → ignore size scoring.

6. Always include size classification in final scoring JSON

AI must output a field such as:

"vessel_size_category": "coaster (4001–17000 DWT)"


This ensures transparency of how the size influences all scoring criteria.
