OPEN AREA (correction to SECTION 1:205):
  The field "open_area" is NOT a string — it is an OBJECT or null:
    {
      "name": "N.SPAIN",
      "start_at": ISODate,
      "end_at": ISODate,
      "updated_at": ISODate,
      "comments": ["Cargo: DRY/PROJECTS/BREAKBULK", ...]
    }

  Usage:
    - Use "name" for P1 proximity check
    - Use "start_at/end_at" for P7 readiness
    - Parse "comments" array via Comments Processing Guide (P6)

TRADING RESTRICTIONS:
  Field: "trading_restrictions" (array)
  Structure: [{geo_name, action, notes}, ...]

  Usage:
    - Use in P8 for geographical blocks
    - "action" values: "block", "require_confirmation", "avoid"
    - Example: {geo_name: "Iran", action: "block"} → cargo to IR port = BLOCK

INTAKES (post-TZ_INTAKE 2026-04-22 — NEW SHAPE):
  Field: "intakes" (array)
  Structure: [
    {
      "dwt_value": int (nullable),
      "cargo": {                         ← single object, NOT array of cargo_types
        "id": int,                       ← cargos_catalog.id
        "display_name": string,          ← "cargo_type - cargo_item" (e.g. "grain - WHEAT")
        "cargo_type": string|null,       ← catalog type (e.g. "grain")
        "cargo_item": string|null,       ← catalog item (e.g. "WHEAT")
        "measurement_mode": "stw"|"dimensions",
        "stw_value": string|null,        ← per-vessel actual STW (e.g. "46" or "54-55")
        "stw_defaults": [string]|null,   ← library defaults from cargos_catalog
        "dimensions": string|null        ← free text when measurement_mode='dimensions'
      } OR null,
      "geo_names": [string],             ← merged ports/regions/countries/seas
      "notes": string|null,
      "created_at": ISODate
    }
  ]

  Usage for P5 intake_capacity:
    Step A — per-vessel actual STW override (HIGHEST PRIORITY):
      If cargo.stw_value is set → stw = cargo.stw_value
        (example "46"; may be range like "54-55" — use midpoint)
        → confidence = "high" (from actual vessel history)
    Step B — library default STW:
      Elif cargo.stw_defaults is non-empty → stw = first of stw_defaults
        → confidence = "medium" (catalog default for this cargo)
    Step C — LLM knowledge fallback:
      Else → stw = LLM estimate (market-typical for cargo_type/cargo_item)
        → confidence = "low", note in reasoning:
          "no vessel-specific STW data, using general market knowledge"
    Then: volume_intake = grain_capacity_cbft / stw

  Usage when measurement_mode == "dimensions":
    → Do NOT apply STW formula (stw_value would be null).
    → Use cargo.dimensions (free text) for project cargo / breakbulk.
    → Compare with vessel hold dimensions from custom_attributes
      (slugs: hold-dimm, hatch-dimm) to assess physical fit.
    → Example: cargo.dimensions = "40x20x10m bags, stackable max 3 high"
      → P5 reasoning: "Using reported cargo dimensions for breakbulk intake;
        no STW applicable (project cargo)."

  Backward compatibility (short transition window):
    → If a legacy document still has intakes[].cargo_types (array of strings)
      and no cargo object → treat as missing STW data, cascade straight to
      Step C (LLM estimate) and note in reasoning that data is pre-TZ_INTAKE.
      This path should disappear after full resync.

CARGO PREFERENCES (post-TZ_INTAKE 2026-04-22 — NEW SECTION):
  Field: "cargo_preferences" (array)
  Structure: [
    {
      "cargo_catalog_id": int,
      "cargo_display_name": string,      ← e.g. "BARLEY" or "grain - SBM"
      "cargo_type": string|null,
      "cargo_item": string|null,
      "preference_setting_id": int,
      "preference_setting_name": string, ← e.g. "Preferred" | "Neutral" | "Blocked" | "Avoid"
      "source": "manual" | "snapshot"    ← manual = per-vessel rule, snapshot = inherited
    }
  ]

  Usage for P3 cargo_preferences scoring:

    PRIORITY RULE (when same cargo_catalog_id appears multiple times):
      source = "manual"   → HIGHEST priority (owner explicitly set this rule)
      source = "snapshot" → MEDIUM priority (inherited group-level rule)
      When both exist for the same cargo → manual overrides snapshot.

    SCORING:
      Current cargo matches CARGO PREFERENCES with "Preferred"
        → +25 points for P3
        → reasoning: "vessel owner marked {cargo} as preferred"
      Current cargo matches with "Blocked" or "Avoid"
        → -30 points
        → decision = "blocked", critical = true
        → reasoning: "vessel owner marked {cargo} as blocked/avoid"
      Current cargo matches with "Neutral"
        → 0 points (no bonus, no penalty)
        → note in reasoning: "cargo flagged as neutral in vessel preferences"
      Current cargo NOT in cargo_preferences
        → 0 points
        → fall back to general market knowledge for P3 scoring
        → reasoning: "no vessel-specific preference, using general knowledge"

    MATCHING RULES (in order of precedence):
      1. Match by cargo_catalog_id exactly when cargo snapshot has cargo_catalog_id.
      2. Match by display_name case-insensitive when no cargo_catalog_id.
      3. Match by cargo_type when preference specifies type-level only
         (cargo_item = null) — this covers rules like "all steels", "all grain".

CUSTOM PROMPTS:
  Fields: "companies[].custom_prompt", "people[].custom_prompt"

  Usage:
    - If company.custom_prompt set → treat as system-level override for this company's offers
    - If person.custom_prompt set → personalize offer text for this person
    - Empty/null → ignore, use standard templates

PERSON LOCATION & NATIONALITY:
  Fields: "people[].location", "people[].nationality"

  Usage for RECIPIENT_LOCAL_TIME (timezone greeting):
    Priority 1: location (actual physical location, e.g. "Istanbul")
    Priority 2: nationality (home country timezone, e.g. "turkish" → Istanbul)
    Priority 3: vessel.flag (vessel country)
    Priority 4: skip timezone hint, default greeting

BIRTHDAY GREETINGS:
  Field: "people[].birthday" (UTCDateTime, day+month only — year=0)

  Usage in offer:
    - If person.birthday month-day matches TODAY (UTC or recipient TZ) → prepend
      one-line greeting: "hbd {first_name}! wishing you well." THEN main body.
    - If birthday is tomorrow → optionally soft: "almost yr bday, great timing!" (rare use)
    - If birthday is null or not today → DO NOT mention, DO NOT fabricate

COMPANY PREFERENCES (computed by Laravel, derived from comments):
  Field: "companies[].preferences" (object)
  Known keys:
    - "no_poc" (bool) — if true, owner refuses POC region cargo (Pivdennyi+Odessa+Chornomorsk)

  Usage:
    - If preferences.no_poc=true AND cargo in POC → P2 BLOCK (stronger than RULE 11 text parsing)
    - If preferences.no_poc=false AND owner comment contains "no odessa" → still apply RULE 11
    - This is an OPTIMIZATION over RULE 11 text parsing (less error-prone)
